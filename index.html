The gap the agents leave behind

Here is how we think about your customers' stack. Most of what an engineering team calls an AI problem is not one. About 60% is traditional code and database work. About 30% is rule-based logic. About 10% is a genuine agent problem. Teams who skip this call end up running LLMs on work Postgres or a rule engine would do faster, cheaper, and with better guarantees.

Duplo's agents are the 10% done right. The gap is the decision framework that tells a customer which work belongs there.

60%

Code and database work

Postgres, queues, deterministic services. No model required.

30%

Rule-based logic

Policy engines, rules, guardrails. Your rules engine pattern lives here.

10%

Genuine agent work
